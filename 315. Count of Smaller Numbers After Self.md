## Problem
Given an integer array `nums`, return _an integer array_ `counts` _where_ `counts[i]` _is the number of smaller elements to the right of_ `nums[i]`.

**Example 1:**

**Input:** nums = [5,2,6,1]
**Output:** [2,1,1,0]
**Explanation:**
To the right of 5 there are **2** smaller elements (2 and 1).
To the right of 2 there is only **1** smaller element (1).
To the right of 6 there is **1** smaller element (1).
To the right of 1 there is **0** smaller element.

**Example 2:**

**Input:** nums = [-1]
**Output:** [0]

**Example 3:**

**Input:** nums = [-1,-1]
**Output:** [0,0]

**Constraints:**

- `1 <= nums.length <= 105`
- `-104 <= nums[i] <= 104`

## Solution - Easy not optimal

To solve this problem we are gonna make use of two extra variables: `result` which is the list to return and `sorted_array` which will be the sorted array as we go over each item.

The way this solution works is the following. We are gonna start iterating over the initial arra `nums` from the back to the front. And as we go inserting that element into `sorted_array` we are gonna capture the position `index` in which it is inserted, because that will mean it has over `index` elements before that are smaller. 

> **Note**: In case there are repetitions, we want to insert each element into their most utmost position. Here we do that with making use of `bisect_left` method from the sorted array library `sortedcontainer: SortedList`

After finishing the loop, we return the `result` array completely flipped.

```python
from sortedcontainers import SortedList

class Solution:
    def countSmaller(self, nums: List[int]) -> List[int]:
        sorted_list = SortedList()
        result = []

        for i in range(len(nums) -1, -1, -1):
            num = nums[i]

            idx = sorted_list.bisect_left(num)
            result.append(idx)
            sorted_list.add(num)

        
        return result[::-1]
```
